제공 형태 — API vs BULK

형태방식적합한 작업
API 실시간REST GET, 응답 XML최신성·소량·온디맨드. 특정 건 조회, 변경 폴링, 실시간 연동
BULK 대량파일기간제 / 선택형 파일전수분석·자체DB 적재·백필. 인용망, 패밀리, AI 코퍼스
🔁
표준 패턴: BULK(선택형)로 과거분 일괄 적재 → API로 변경분 실시간 동기화. "BULK로 적재 → API로 최신화."

도메인 그룹별 분석

📰 공보 원천 1차
특허청 발행 공식 공보 원문. 권리 발생/변경 공시. 신규 출원·등록 모니터링, 자사 DB 피드 적재. (예: 특허·실용 공개·등록공보, 디자인 공보, 상표 출원 속보, 정정공보)
📋 행정처리 절차 가공
출원~등록~소멸 심사 절차 타임라인. 행정처리 이력·통지서 마감기한·청구항/분류 변동. 기한관리(데드라인 알림), 심사상태 대시보드. TM5 공통상태지표는 한·미·유·일·중 표준 상태아이콘.
⚖️ 심판
특허심판원 단계. 무효·거절불복·권리범위확인·정정 심판의 청구·결과(인용/기각/각하). 분쟁 리스크 모니터링, 소송 전 IP 평가.
🧬 공통 — 법적상태·등록·결정서 권종 표준
권종을 가로질러 표준화한 척추 데이터. 법적상태 이력은 WIPO 국제표준 매핑(특허·실용 ST.27, 디자인 ST.87) → 권리 생존성(alive/dead) 판정. 등록사항·의견제출통지서·거절/등록결정서·분류코드.
🔗 부가 — 인용·패밀리·서열 분석 파생
서지에서 한 단계 더 추출. 인용/피인용 네트워크, 패밀리(국가별 대응출원), 염기서열·합금조성비·기탁 미생물. 핵심특허 식별, 글로벌 출원전략, 바이오·소재 선행조사.
👤 인명 — 출원인/권리자
출원인·권리자 엔티티 정규화. 명칭 변동·대표 출원인·법인 매칭·권리자 변동·기술분야. 동일 출원인 통합(entity resolution), 기업별 포트폴리오 집계.
📚 기타 — 사전·아이디어
보조·사전 데이터. 시소러스(동의어 → 검색어 확장으로 재현율↑), 의약품 국제일반명칭(INN), 상표 원산지 명칭, 인터넷 기술공지(방어공개).
🤖 AI 학습데이터
ML 학습용 정제 코퍼스. 기계번역용 국문초록, 한국특허영문초록(KPA), 특허 중한 코퍼스(병렬말뭉치). 특허 NLP·기계번역 모델 학습 — 사실상 BULK 전용.
🌐 해외 IP데이터 실시간 중계
해외특허/상표/디자인 + 공통. 47~50은 API 전용 → 외부 IPO를 실시간 룩업 게이트웨이로 중계하는 구조. 글로벌 선행조사, 해외출원 전 FTO.

인증·호출

같은 KIPRIS 데이터라도 어디서 키를 받았는지에 따라 키 종류·파라미터명·베이스 URL이 다르다.

📥 공공데이터포털 무료·쉬움
data.go.kr · 데이터셋별 활용신청(다수 자동승인, 즉시) · 키 = serviceKey (Encoding/Decoding 2종) · 프로토타입·소량에 적합
🏛️ KIPRIS Plus 전체·대량
plus.kipris.or.kr · 회원가입→서비스 신청(무료 즉시 사용 — 실사용에서 형식 승인 관측 이력 있음) · 키 = accessKey · 무료+유료+BULK 전체. 상용·대량 연동
🔑
data.go.kr = KIPRIS Plus 무료분 일부를 쉽게 노출한 창구. KIPRIS Plus = 슈퍼셋. 코드에서 키 분기 필요.
KIPRIS Plus 호출 예시 (accessKey · 검증된 실제 엔드포인트)
# 출원번호 검색 http://plus.kipris.or.kr/openapi/rest/patUtiModInfoSearchSevice/applicationNumberSearchInfo ?applicationNumber=1020200012345&patent=true&utility=true &docsStart=1&docsCount=10&sortSpec=AD&accessKey=발급키 # 자유검색 http://plus.kipris.or.kr/openapi/rest/patUtiModInfoSearchSevice/freeSearchInfo ?word=드론&patent=true&pageNo=1&numOfRows=10&accessKey=발급키
data.go.kr 호출 예시 (serviceKey)
http://apis.data.go.kr/.../서비스명?serviceKey=인코딩된키 &numOfRows=10&pageNo=1&applicationNumber=1020200012345

응답 포맷 & 상태코드

기본 XML — response > header(resultCode/resultMsg) > body > items > item. JSON은 서비스별 지원 여부를 명세서에서 개별 확인(KIPRIS 계열은 XML이 사실상 기본). 페이징 pageNo/numOfRows 또는 docsStart/docsCount.

resultCode의미
00정상
10요청 파라미터 오류
20결과 없음 — 에러 아님, 빈 배열 방어처리
30accessKey 미등록
31사용기한 만료

요금

  • 무료 — 월 1,000회 (data.go.kr 또는 KIPRIS Plus 무료 신청)
  • 유료 Open API — 1상품 일 5,320원 × 365 ≈ 연 194만원 (일 단위 기간 지정 가능)
  • 2종 이하 신청 50% 할인 · 개인·중소기업·공공·비영리 추가 50% 할인(중복 가능)

실무 함정 ⚠️

  • 출원번호 하이픈 제거 — 10-2020-0012345 → 1020200012345로 전달. 응답은 하이픈 포함 가능 → 표시/검색 분리.
  • serviceKey 이중 인코딩 (data.go.kr 최다 오류 SERVICE_KEY_IS_NOT_REGISTERED_ERROR) — Encoding 키의 %2B를 URL 빌더가 %25로 재인코딩하면 깨짐. 해결: URI 객체로 직접 구성(build(true).toUri()).
  • 엔드포인트 철자 — patUtiModInfoSearchSevice (Service 아님, Sevice) 그대로.
  • 일자 포맷 — YYYYMMDD 8자리, 구분자 없음.
  • HTTP/HTTPS — KIPRIS Plus 직접은 http 예시가 흔함. 운영 전 https 확인.
